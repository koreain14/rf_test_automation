Summary
- Adds execution-model foundation files to the project as first-class source files.
- Adds safe bridge recording from RunServiceStep into step_results without changing final verdict logic.
- Adds instrument-profile resolver and path resolver foundations for Keysight N9020/N9030, R&S FSW, TMX/HCT Switch, and E3632A expansion.
- Keeps existing Preset Editor / WLAN Expansion / Add Plan / Run flow intact.

3) Stabilization update
   - application/run_service_step.py updated to record EXECUTION_MODEL / EXECUTOR_PREVIEW in a non-fatal way.

Important notes
- This patch does NOT replace the current procedure-based run path.
- Final PASS/FAIL/SKIP/ERROR logic remains on the existing procedure path.
- EXECUTION_MODEL and EXECUTOR_PREVIEW are metadata bridges for the next phase.
- Plan Editor feature work is NOT included in this patch. Current Plan tab remains as-is.
  Reason: this release focuses on run-path stabilization and instrument-layer foundations, to minimize impact.

Recommended checks
1. Launch app.
2. Create/import expansion-based preset.
3. Add Plan.
4. Run.
5. Confirm results are created and final verdict logic remains unchanged.
6. Open result step details and verify EXECUTION_MODEL / EXECUTOR_PREVIEW rows exist.
